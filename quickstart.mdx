Get your mango damage detection system up and running quickly with Docker. This guide will walk you through deploying both the Django web interface and FastAPI prediction service.

## Prerequisites

Before you begin, ensure you have:

- [Docker](https://www.docker.com/get-started) installed and running
- [Git](https://git-scm.com/) for cloning the repository
- At least 2GB of free disk space for the model and containers

<Note>
This quickstart uses Docker Compose to run both the Django web interface (port 8000) and FastAPI prediction service (port 8001) in separate containers.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Mango Damage Detection repository to your local machine:

    ```bash
    git clone https://github.com/your-username/mango_app.git
    cd mango_app
    ```
  </Step>

  <Step title="Download the pre-trained model">
    Download the TensorFlow model and place it in the correct directory:

    ```bash
    # Create the model directory if it doesn't exist
    mkdir -p fastapi_app/model

    # Download the model (alternatively, download manually from https://bit.ly/pyconmodelh5)
    curl -L https://bit.ly/pyconmodelh5 -o fastapi_app/model/mango_model.h5
    ```

    <Warning>
    The model file (`mango_model.h5`) must be placed in `fastapi_app/model/` before building. The FastAPI service will fail to start without it.
    </Warning>
  </Step>

  <Step title="Build the Docker containers">
    Build both the Django and FastAPI images:

    ```bash
    docker compose build
    ```

    This creates two images:
    - `mango_app`: Django web interface
    - `mango_fastapi`: FastAPI prediction service
  </Step>

  <Step title="Start the services">
    Launch both containers:

    ```bash
    docker compose up
    ```

    <Tip>
    Add the `-d` flag to run in detached mode: `docker compose up -d`
    </Tip>

    Wait for both services to start. You should see output indicating:
    - Django running on `0.0.0.0:8000`
    - FastAPI (Uvicorn) running on `0.0.0.0:8001`
  </Step>
</Steps>

## Make Your First Prediction

### Using the Web Interface

1. Open your browser and navigate to `http://localhost:8000`
2. Click the file upload button and select a mango image
3. Click "Submit" to analyze the image
4. View the prediction result showing the damage type

<Note>
The model classifies mangoes into five categories: Anthracnose, Bacterial-Black-spot, Fruitly, Healthy-mango, and Others.
</Note>

For detailed guidance on using the web interface, see the [Web Interface Guide](/guides/web-interface).

### Using the API

You can also make predictions directly via the FastAPI endpoint:

<CodeGroup>
```bash curl
curl -X POST "http://localhost:8001/predict/" \
  -H "accept: application/json" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/your/mango_image.jpg"
```

```python Python
import requests

url = "http://localhost:8001/predict/"
files = {"file": open("mango_image.jpg", "rb")}

response = requests.post(url, files=files)
print(response.json())
```

```javascript JavaScript
const formData = new FormData();
formData.append('file', fileInput.files[0]);

fetch('http://localhost:8001/predict/', {
  method: 'POST',
  body: formData
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));
```
</CodeGroup>

**Example Response:**

```json
{
  "prediction": "Healthy-mango"
}
```

The API returns one of five damage classifications:
- `Anthracnose` - Fungal disease causing dark lesions
- `Bacterial-Black-spot` - Bacterial infection with black spots
- `Fruitly` - Fruit fly damage
- `Healthy-mango` - No visible damage
- `Others` - Other types of damage

## Understanding the Architecture

The application runs two separate services that communicate over a Docker network:

- **Django (port 8000)**: Handles the web interface and user file uploads
- **FastAPI (port 8001)**: Serves the TensorFlow model and processes predictions

When you upload an image through the web interface, Django forwards it to FastAPI's `/predict/` endpoint, which processes it through the model and returns the classification.

<Tip>
The services communicate via the `mango_network` Docker network. Django uses `http://fastapi:8001` internally to call the FastAPI service.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If ports 8000 or 8001 are already in use, modify the port mappings in `docker-compose.yml`:

    ```yaml
    ports:
      - "8002:8000"  # Change host port to 8002
    ```
  </Accordion>

  <Accordion title="Model file not found">
    Ensure the model file is at `fastapi_app/model/mango_model.h5`. Check the file exists:

    ```bash
    ls -lh fastapi_app/model/mango_model.h5
    ```

    If missing, re-download from [https://bit.ly/pyconmodelh5](https://bit.ly/pyconmodelh5)
  </Accordion>

  <Accordion title="Connection refused between services">
    If Django cannot connect to FastAPI, ensure both containers are on the same network:

    ```bash
    docker network inspect mango_network
    ```

    Both `mango_app` and `mango_fastapi` containers should be listed.
  </Accordion>
</AccordionGroup>

## Stopping the Services

To stop the running containers:

```bash
# Stop containers but keep them
docker compose stop

# Stop and remove containers
docker compose down

# Stop, remove containers, and clean up volumes
docker compose down -v
```
