---

Run the Mango Damage Detection application locally for development. This setup requires running two separate processes: Django for the web interface and FastAPI for the model server.

## Requirements

Before starting, ensure you have:

- Python 3.8 or above (Python 3.12 recommended)
- pip (Python package manager)

## Dependencies

The application requires the following packages from `requirements.txt`:

```txt requirements.txt
django
fastapi
uvicorn
tensorflow
python-multipart
pillow
numpy
requests
```

## Installation

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/your-username/mango_app.git
    cd mango_app
    ```
  </Step>

  <Step title="Create and Activate Virtual Environment">
    <CodeGroup>
    ```bash Unix/Linux/MacOS
    python3 -m venv django-env
    source django-env/bin/activate
    ```

    ```bash Windows
    python -m venv django-env
    django-env\Scripts\activate
    ```
    </CodeGroup>

    <Tip>
    Your terminal prompt will change to show `(django-env)` when the virtual environment is active.
    </Tip>
  </Step>

  <Step title="Install Dependencies">
    Install all required packages:

    ```bash
    pip install -r requirements.txt
    ```

    This installs Django, FastAPI, TensorFlow, and all other dependencies.
  </Step>

  <Step title="Download the TensorFlow Model">
    <Warning>
      The pre-trained model file is required and must be downloaded separately.
    </Warning>

    Download the model from [bit.ly/pyconmodelh5](https://bit.ly/pyconmodelh5) and place it in the correct location:

    ```bash
    # Create the model directory
    mkdir -p fastapi_app/model/
    
    # Place the downloaded mango_model.h5 file here
    # Final path: fastapi_app/model/mango_model.h5
    ```
  </Step>

  <Step title="Set Up Django Database">
    Run migrations to set up the SQLite database:

    ```bash
    python manage.py migrate
    ```

    This creates the `db.sqlite3` database file in the project root.
  </Step>
</Steps>

## Running the Application

<Warning>
You need to run **TWO separate processes** simultaneously: one for Django and one for FastAPI.
</Warning>

### Terminal 1: Start Django Server

In your first terminal window, with the virtual environment activated:

```bash
python manage.py runserver
```

The Django server will start on **port 8000**:

```
Starting development server at http://127.0.0.1:8000/
```

### Terminal 2: Start FastAPI Server

Open a **second terminal window**, navigate to the project directory, activate the virtual environment, and run:

```bash
uvicorn fastapi_app.api:app --reload --port 8001
```

The FastAPI server will start on **port 8001**:

```
INFO:     Uvicorn running on http://127.0.0.1:8001
```

<Note>
The `--reload` flag enables auto-reloading when code changes are detected, useful for development.
</Note>

## Port Configuration

| Service | Port | URL |
|---------|------|-----|
| Django | 8000 | http://127.0.0.1:8000/ |
| FastAPI | 8001 | http://127.0.0.1:8001/ |

### Changing Ports

If you need to use different ports:

**Django**: Specify the port when running:
```bash
python manage.py runserver 8080
```

**FastAPI**: Use the `--port` flag:
```bash
uvicorn fastapi_app.api:app --reload --port 8002
```

## Accessing the Application

Once both servers are running:

- **Web Interface**: Open [http://localhost:8000/](http://localhost:8000/) in your browser
- **FastAPI Docs**: View the interactive API documentation at [http://localhost:8001/docs](http://localhost:8001/docs)
- **Prediction Endpoint**: POST requests to [http://localhost:8001/predict/](http://localhost:8001/predict/)

## API Endpoints

### Django Endpoints

- `GET /` - Home page for uploading mango images

### FastAPI Endpoints

- `POST /predict/` - Submit an image for damage detection

### Testing the API

Test the prediction endpoint using curl:

```bash
curl -X POST "http://localhost:8001/predict/" \
  -H "accept: application/json" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@path/to/mango_image.jpg"
```

## Development Workflow

### Making Changes

1. **Django Changes**: Modify files in the main project directory
   - Changes are automatically detected with `runserver`
   - View changes at http://localhost:8000/

2. **FastAPI Changes**: Modify files in `fastapi_app/`
   - Changes are automatically reloaded with `--reload` flag
   - View API docs at http://localhost:8001/docs

### Database Migrations

After making model changes:

```bash
python manage.py makemigrations
python manage.py migrate
```

### Creating a Superuser

To access the Django admin interface:

```bash
python manage.py createsuperuser
```

Then visit http://localhost:8000/admin/

## Troubleshooting

### Port Already in Use

If you see `Error: That port is already in use`:

1. Find and stop the process using the port, or
2. Run the server on a different port (see Port Configuration above)

### Module Not Found Errors

Ensure your virtual environment is activated:

```bash
# You should see (django-env) in your prompt
# If not, activate it:
source django-env/bin/activate  # Unix/Linux/MacOS
django-env\Scripts\activate      # Windows
```

### Model Loading Errors

<Warning>
If FastAPI fails to start or shows model loading errors, verify that `mango_model.h5` exists at `fastapi_app/model/mango_model.h5`.
</Warning>

### Django Can't Connect to FastAPI

When running locally, Django connects to FastAPI at `http://localhost:8001`. If you changed FastAPI's port, update the connection settings accordingly.
